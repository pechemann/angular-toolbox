Dialog API

Since v 1.4.6 Experimental

The Dialog API allows to create modal or non-modal dialog box or other interactive component, such as a dismissible alert, inspector, or subwindow.
This component is part of the LAF-less API.

DialogService Class

The DialogService class provides the API for managing injection of customs component into the HTML dialog element created by the ATX framework.

DialogService.show()

show<T>(compRef: any, config: DialogConfig | null = null): ComponentRef<T>

Creates an instance of the type of the compRef parameter, adds this instance to the HTML dialog element and displays the dialog element in front of all other components in the application.

Parameters
compRef: any

The class that represents the component to display as a popup element.

config: DialogConfig

The dialog element config associated with the new component instance to display.
Default value is null.

Returns
ComponentRef<T>

The new ComponentRef onject which contains the component instance and the host view.

Exeptions
DialogServiceError

Thrown if no <atx-dialog-outlet/> tag has been declared whithin the application.

HttpMockService.hide()

hide(): void

Indicates to the ATX framework to remove component instances previously created with the show() method.

Events

HttpMockService.dialogStateChange

readonly dialogStateChange: EventEmitter<DialogOutletEvent>

Dispatches an event that indicates the state of the dialog element when a custom component is added to, or remove from it.
See the DialogOutletEvent API for details.

Destroyable Interface

The Destroyable interface provides implementators the capability to destroy their contents.

Since TypeScript interfaces are never compiled, developers should always take advantage of adding such an interferface in their projects.
The implementation of a destruction mechanism can be easily tested to ensure the destruction of objects and the proper functioning of your application.

Destroyable.destroy()

destroy(): void | any

The destroy() must contain mechanisms to delete all internal references. That will make the object instance eligible for garbage collection.

Returns
void | any

The destroy() usually returns void.
Other types may possible, depending on each implementation.

Angular Toolbox Logging Framework

The ATX Logging Framework provides an easy-to-use and highly configurable solution to add a logging system to your Angular libraries and applications.

Since v 0.12.0

API

Interface Description Link
Logger The base interface of all logging services in the ATX logging framework.
Log Represents a log in the Angular Toolbox framework.
TransactionalLogger Transactional loggers allow to create proxy via the LogConnector interface to send logs to third-party tools.
LogConnector The markup interface for Angular Toolbox log connectors.
LoggerConfig Defines properties for the LoggerService singleton configuration.
Utility Description Link
LogBuilder A static builder that creates new Log objects.
LogUtil A convenient utility class for formatting logs.
LogLevel Defines the level of an Angular Toolbox log.
Service Description Link
LoggerService A convenient high-level singleton that extends the AbstractLogger class.
AbstractLogger The base class for all Logger objects.
Connector Description Link
DefaultLogConnector The default ATX log connector. Allows to process logs silently.
This can be useful for removing logs from production environments, if needed.
ConsoleLogConnector An implementation of the LogConnector interface that sends logs to the JavaScript native console object.
HtmlLogConnector An implementation of the LogConnector interface that processes logs as HTML renderable elements.
Provider Description Link
DEFAULT_LOG_CONNECTOR A convenient immutable instance of the DefaultLogConnector class.
ATX_LOGGER_CONFIG The default provider for the LoggerService singleton configuration.
Constant Description Link
ATX constant strings A list of constants used as core references by the ATX Logging Framework.

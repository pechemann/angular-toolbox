HTTP Mocking Framework

The HTTP Mocking Framework provides an easy-to-use and highly configurable solution to mock HTTP responses to API calls sent from your Angular applications.

Loggers

Since v 0.12.2

Since the HTTP calls are intercepted by the HTTP Mocking Framework, it is not possible to access HTTP logs in the browser debug console. However, the framework provides HTTP logs built on top of the ATX Logging Framework.

To access the HTTP logs, simply associate a connector with the HttpMockLoggingService singleton. The following example shows how to configure the HTTP Mocking Framework for sending HTTP logs to the browser console:

ATX Monitoring Console

Since v 0.12.3

The HTTP Mocking Framework provides a component that allows to easily inspect network activity intercepted by the framework: the ATX Monitoring Console.

You can open this component within a popup window by using the AtxHttpMockConsoleService singleton:

The most common use cases for the ATX Monitoring Console are:

  • Making sure that resources are actually being uploaded or downloaded at all.
  • Inspecting the properties of an individual resource, such as its HTTP headers, content, size, and so on.

It allows Developer to focus on business code, instead of wasting time with invalid data sources, or multi-environment issues (eg. PHP, Electron Apps, JAVA, etc.). Hence, docked version of the ATX Monitoring Console significantly increases productivity.

HTTP Mocking Framework

The HTTP Mocking Framework provides an easy-to-use and highly configurable solution to mock HTTP responses to API calls sent from your Angular applications.

HttpMockInterceptor Interface

HttpMockInterceptor objects represent the configuration of several mocking strategies for HTTP calls.

HttpMockInterceptor.id

id: string

The ID of this interceptor.
It is a best practice to use human-readable values related to the API.
For example, the ID related to the "/todos/:id" route should be "todos".

HttpMockInterceptor.origin

origin?: string

An optional string that represents the URL origin for this interceptor, as defined by the URL standard.
This value redefines the origin specified by the HttpMockConfig object, when it exists.

If both, HttpMockConfig.origin HttpMockInterceptor.origin properties are omitted, the framework uses the current app URL to determine the global context of HTTP call interceptions.
See the Origin section of the URL specification for details.

HttpMockInterceptor.endpoints

endpoints: HttpMockEndpoint[]

A list of endpoints that must be intercepted by the HTTP Mocking Framework.
See the HttpMockEndpoint interface reference for details.

HttpMockInterceptor.description

Since v 1.4.1 
description?: string

The description of this interceptor.
This property is optional.

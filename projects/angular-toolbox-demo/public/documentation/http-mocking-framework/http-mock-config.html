HTTP Mocking Framework

The HTTP Mocking Framework provides an easy-to-use and highly configurable solution to mock HTTP responses to API calls sent from your Angular applications.

HttpMockConfig Interface

HttpMockConfig objects represent the configuration of several mocking strategies for HTTP calls.

HttpMockConfig.origin

origin?: string

An optional string that represents the origin of all endpoints that are intercepted.
This value is redefined by the HttpMockInterceptor.origin property, when it exists.

If omitted, the framework uses the current app URL to determine the global context of HTTP call interceptions.
See the Origin section of the URL specification for details.

HttpMockConfig.id

id?: Uuid

An optional string that represents the unique identifier for this configuration object.
See the Uuid class reference for details.

HttpMockConfig.interceptors

interceptors: HttpMockInterceptor[]

A collection of specific mocking strategies for each API.
See the HttpMockInterceptor interface reference for details.

HttpMockConfig.description

Since v 1.4.1 
description?: string

The description of the API associated with this HttpMockConfig objects. (Supports rendering of HTML tags.)
This property is optional.

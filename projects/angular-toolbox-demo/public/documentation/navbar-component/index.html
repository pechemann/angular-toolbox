ATX Navbar

Since v 1.5.3 Experimental

Navbar components are responsive navigation headers that include support for branding, navigation and mobile collapsing menu.
This component is part of the LAF-less API.

Principle

Regarding the LAF-less API, navbar components allows to create responsive navigation menus by separating the UI logic from the app business logic.

Navigation bars provide a standard structure for interaction on a modern website. Following this structure, the ATX Navbar component is divided into 3 parts:

Brand
This container allows branding information to be displayed even if the menu is collapsed in responsive mode. You must identify the brand content by using the special brand tag.
Burger
The Burger section contains the toggle button which allows to expand, or collapse, the main menu in responsive mode. It is hidden in standard mode. The ATX framework identifies the toggle button icon by referring to the special icon tag.
Menu
This container renders navigation links declared by developers. All navigation links should be wrapped within LI elements, since they are direct children of an UL element.
Any element that is not identified by the brand or icon tags is rendered as a navigation menu item.

API

Component Description Link
NavbarComponent A horizontal menu that can be used to show a list of navigation links positioned on the top side of your page.

Styles

ATX Navbar component has been designed to let you implement your own CSS styles. The following table shows all the style declarations you can use for this purpose.

Class Name Description Aria Role
atx-navbar The class name of the root NAV element. menubar
atx-menu-open The class added to the root NAV element when the menu is open. ---
atx-menu-action The class name of the DIV element that contains both, brand container and toggle button. ---
atx-brand The class name of the DIV element that contains brand information. ---
atx-burger The class name of the toggle button container. button
atx-menu The class name of the UL element that contains all menu items. menu

You typically use the ::ng-deep pseudo declaration to override the rules above.

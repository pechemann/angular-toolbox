Version Service

A lightweight service that provides Semantic Versioning implementation for your Angular apps.

Motivation

According to Semantic Versioning 2.0.0:

[Semantic Versioning proposes] a simple set of rules and requirements that dictate how version numbers are assigned and incremented.

Providing users the version en the build date of an app is a classical functionality in software development. Since HTML-based single page application are becoming a major standard in the software industry, it is important to provide such information to users.

The purpose of the Angular Toolbox version service is to allow Developer to easily implement Semantic Versioning in their Angular apps.

API

Interface Description Link
Version Specifies the semantic versioning of an Angular application.
VersionConfig Provides the basic API that defines a version config token.
VersionManager The interface implemented by services that specifie the semantic versioning of an Angular application.
Class Description Link
AbstractVersionManager The abstract class for all services that implement the VersionManager interface.
Provider Description Link
VERSION_CONFIG The default provider for the VersionService class configuration.
Utility Description Link
VersionUtil The default provider for the VersionService class configuration.

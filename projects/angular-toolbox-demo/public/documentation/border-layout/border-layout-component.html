Border Layout

Since v 1.3.0

A border layout lays out a container, arranging and resizing its components to fit in five regions: north, south, east, west, and center.

BorderLayout Component

BorderLayout components are useful for building software where you need to expose graphic tools to allow the user to perform complex actions.

Each region of a BorderLayout component is defined by a BorderLayoutContainer container; regions are identified by a corresponding LayoutRegion constant: NORTH, SOUTH, EAST, WEST, and CENTER:

Each region can be enlarged using a handle, depending on the parameters defined by the developer.

Events

BorderLayout.dragStart

readonly dragStart: EventEmitter<LayoutDragEvent>

Emits events each time the user starts dragging a region handle.

BorderLayout.dragStop

readonly dragStop: EventEmitter<LayoutDragEvent>

Emits events each time the user stops dragging a region handle.
See the LayoutDragEvent API for details.

BorderLayout.dragging

readonly dragging: EventEmitter<LayoutDragEvent>

Emits events each time the user is dragging a region handle.

Border Layout

Since v 1.3.0

A border layout lays out a container, arranging and resizing its components to fit in five regions: north, south, east, west, and center.

LayoutConstraints Interface

Defines the layout constraints of a BorderLayoutContainer object.

LayoutConstraints.region

region: LayoutRegion

Specifies the region associated with a BorderLayoutContainer object.
See the LayoutRegion enum API for details.

LayoutConstraints.resizable

resizable?: boolean

Indicates whether the BorderLayoutContainer object can be resized (true), or not (false).
The BorderLayoutContainer object is not resizable by default.

LayoutConstraints.size

size?: number

Defines the default size of a BorderLayoutContainer object.
The default size of a BorderLayoutContainer object depends on its region:

  • LayoutRegion.NORTH, LayoutRegion.SOUTH: default size is 50 pixels
  • LayoutRegion.EAST, LayoutRegion.WEST: default size is 100 pixels

The size value is not applied to containers of type LayoutRegion.NORTH.

LayoutConstraints.minSize

minSize?: number

Defines the minimum size of the BorderLayoutContainer object.
No minimum size is applied by default.

LayoutConstraints.maxSize

maxSize?: number

Defines the maximum size of the BorderLayoutContainer object.
No maximum size is applied by default.

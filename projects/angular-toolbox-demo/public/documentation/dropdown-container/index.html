Dropdown Container

Since v 1.3.2

Dropdown containers allow Users to toggle the display of a specific content by clicking a button.
This component is part of the LAF-less API.

Principle

Regarding the LAF-less API, dropdown containers allows to create non-modal popup elements by separating the popup UI logic from the app business logic.

You typically use dropdown containers to create popup menus, complex select action components and so on.

The dropdown API is built on top of the popover HTML API. Since the popover HTML API is partially implemented by the browser ecosystem, some functionalities are not fully efficient yet. The final version of the dropdown API will allow to display content over all HTML elements. This behavior is the same as the dropdown behavior of native OS.

The current version works the same as all other JavaScript/CSS frameworks. Final version will be available according to the CSS anchor positioning support for all major browsers.

API

Component Description Link
Dropdown Creates popup containers that are displayed by clicking on a button.
Utility Description Link
DropdownState Represents the state of a dropdown container.
DropdownVerticalPosition Defines the content vertical position of a dropdown component.
DropdownHorizontalPosition Defines the content horizontal position of a dropdown component.
--atx-dropdown-gutter Adjusts the space between the button element and the content container.
Event Description Link
DropdownEvent An event that represents a user interaction with a Dropdown component.
DropdownEventType Defines the type of a DropdownEvent instance.
